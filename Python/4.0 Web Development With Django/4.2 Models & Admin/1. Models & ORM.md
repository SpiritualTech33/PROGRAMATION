Okay, these are two fundamental concepts in web development, because this is where we are going to store our data. What are Models and ORM?

Honestly, I don’t want to overcomplicate things. I think the best approach is to keep everything simple, because if you can’t explain something in a simple way, it means you don’t truly understand it.

A model, in general terms, is a Python class. When you create the class, you’re essentially creating a table. In the lines that follow, you describe the columns it will have, and what type of data each column will carry — including what kind of information it will contain and what type of data it accepts.

A model in the context of Django is similar to a table in SQL. You choose the name of each column and determine what type of data it holds. The great advantage of using models is that everything gets stored in a database, and you don’t need to write a single line of SQL. You can do everything using Python syntax. It is the definition of what you want to save. For example, if I declare a model called Prices, Django will create a table called prices, which will contain the columns I define in the model.

Models are Django’s way of saying: “Don’t worry about speaking SQL, you can keep speaking Python and we’ll translate for you.” This is where the ORM comes in. We could say that the ORM is like an intermediary between Python and the database. It takes your Python classes (models) and transforms them into relational databases like SQL, without you having to write a single line of SQL code.

I will show 2 examples of a model, one simple and one more complex. You’ll see that there are many things like models.TextField, models.FloatField, and so on. It doesn’t matter if you still don’t fully understand them yet — we’ll cover that in the next note. Just keep in mind the idea that it’s very similar to the SQL declaration about the type of data that each column carries.

**SIMPLE EXAMPLE**
```python
from django.db import models

class Task(models.Model):
	#Attributes = Data Base Columns
	title = models.CharField(max_length=100) #Short text
	description = models.TextField(blank=True) #Long Text Optional
	is_completed = models.BooleanField(default=False)
	created_at = models.DateTimeField(auto_now_add=True) # Auto timestamp
	

```

As we can see, now we have a table created with the Task name. It has the columns title, description, is_completed and created_at, everything writed on python, it's not a gift?

**COMPLEX EXAMPLE**
```python
# myapp/models.py
from django.db import models
from django.utils import timezone

class Review(models.Model):
    """A user review for a book."""
    book = models.ForeignKey(
        'Book', 
        on_delete=models.CASCADE,
        related_name='reviews'
    )
    reviewer_name = models.CharField(max_length=100)
    rating = models.IntegerField()          # 1 to 5
    comment = models.TextField(blank=True)
    created_at = models.DateTimeField(default=timezone.now)

    def __str__(self):
        return f"Review for {self.book.title} - {self.rating} stars"



```
