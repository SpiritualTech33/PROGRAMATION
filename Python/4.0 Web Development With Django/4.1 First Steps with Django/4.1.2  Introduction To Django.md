**Okay, now that we know what the Web is, we are ready to learn how to use a framework written in Python to build our own website. But before we move forward, I want to dissect those key terms and explore the origins of the framework.**

**Note to myself:** I have noticed that when you investigate how something was born, you understand its true reason for existing much better. For example, when I learned SQLite, remembering that it was created for submarines—so they could have a database that didn’t depend on any server connection—helped me truly grasp the deep purpose of local databases.

**Well, Django was born in a newsroom: The Lawrence Journal-World.** There, journalists needed to publish stories quickly and without mistakes. They couldn’t afford to rebuild user authentication from scratch every single time. So they decided to create Django — a framework written in Python that gives you a complete skeleton for a website right from the start. This way, you can focus only on what really matters instead of spending weeks on configuration.

Look at everything Django handles for you from day one:

- **Ready-to-use user authentication** (login, logout, registration, password reset, permissions)
- **Powerful and customizable admin system** (quite possibly Django’s greatest gift)
- **Form management and validation** (with automatic protection against attacks)
- **Static file handling and file uploads**
- **Internationalization system** (so your app can speak Spanish, English, or any language)
- **Sessions, messaging, caching, and standard web utilities**

**Let’s dissect the terms even further.**

**Framework:** This word reveals so much. It means “a ready-to-use structure.” Imagine you want to build a house but have no idea how. You would need to hire workers, architects to design it, engineers to do the calculations, and someone to buy all the materials. Months would pass before you could even step inside your own bedroom.

Django is like skipping all of that and receiving a house that already has doors, windows, plumbing, and electricity installed. You only need to add the final wiring and the beautiful finishes. That is Django: it takes care of the heavy lifting so you don’t have to worry about it. It gives your web application a solid skeleton that works properly, freeing you to create the real magic.

**Written in Python:** Here comes the best part. Django is written in Python, so you don’t have to switch languages. You can build a complete website using the same friendly, readable Python you already know — with its simple syntax that almost reads like English.

**Django is the translator between the raw HTTP conversation of the Web and the Python code you write.**

text

```
THE WEB (HTTP — the language of request and response)
         ↕
     DJANGO (the framework that speaks HTTP, organized in Python)
         ↕
       PYTHON (the language Django is built in)
         ↕
         YOU (the engineer/philosopher building on top of all three)
```

Django also follows the “batteries included” philosophy. That means you don’t need a thousand extra extensions, services, or libraries just to get started. You simply install Django from your terminal and begin working. Isn’t that an incredible gift?
