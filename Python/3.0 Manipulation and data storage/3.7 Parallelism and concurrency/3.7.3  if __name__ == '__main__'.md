Okey, this is kind of a weird concept, but very important under the parallelism context.
it's very likely that you have been seen this line of code before:
```python
if __name__ == '__main__':
	# Some code here
```

It's very common to see it, but, what does that mean?
When you create a new file, Python internally creates a built in variable called name (with double under score) this variable answer a single question, how is this file beign used right now? 
if the file is executed directly, the variable name == main
```python

python my_script.py

__name__ == "__main__"
```

In other hand, if the file is imported, this variable is gonna have the name of the file
```python

import my_script

__name__ == "my_script"
```

So, with that said, what really this line means? It's saying only run the code block below if this file is being executed directly. 

Why this is important in multiprocessing? well, when you spawn a new process with multiprocessing module, python imports your script from scratch in each new process. 

```python
"I need a new process. Let me start a fresh Python interpreter
and import the original script to set up the environment."
```

If you don't have the `if __name__ == '__main__':` guard, **that import will trigger your process-spawning code again.** Which spawns more processes. Which import the script again. Which spawn more processes.

**Infinite recursion. Your computer catches fire. 🔥**
