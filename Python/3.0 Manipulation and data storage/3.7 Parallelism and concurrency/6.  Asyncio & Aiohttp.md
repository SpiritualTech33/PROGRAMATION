Before diving deep in this 2 libraries, let's think of them like the relation between a motor and a car. Asyncio is a module that allows Python to think asynchronously, and aiohttp gives uses that capability to talk with the web, so are fundamentally 2 modules designed to work asynchronously wit the web, the vast world of the internet.

asyncio  →  The ENGINE of async Python
aiohttp  →  A CAR built on top of that engine

`asyncio` manages time.  
`aiohttp` manages web communication.

In general terms, asyncio is the loop event that you learned before, is that limitless patience coordinator that is always asking for pending tasks, asyncio is the foundation of asynchronous programing in python. And aiohttp is the vehicle witch we can ride through the sea of data from the web.

## 🔧 What Does `asyncio` Give You?

Core tools include:

- `async def` → define coroutine
- `await` → suspend execution
- `asyncio.run()` → start event loop
- `asyncio.create_task()` → schedule concurrent work
- `asyncio.sleep()` → non-blocking sleep


## aiohttp - Async HTTP on Top of asyncio

This is the third party library with asyncio, actually, is built in top of asyncio.
It provides:

- Asynchronous HTTP client
- Asynchronous web server

It uses `asyncio` internally. If `asyncio` is the engine,  `aiohttp` is a vehicle built on that engine.
