# The Four Critical Differences between processes and threads.

### 1. Memory - Isolated vs Shared

THREADS share memory - Dangerous, but fast communication.
If thread A changes a variables, thread B notice it immediately.
This is powerful, and dangerous because of the race conditions.
And with that said, why this is important?
Imagine two writer writing in the same book. Is efficient and fast, but it's very likely that one writer override the other. That's shared memory.
 on the other side, Isolated memory it's like giving a different book te each writer. Safer, but they need to communicate to exchange info. 


### 2. Creation cost - Light weight vs Heavy

Threads are simpler to create, they are only citizens in a new country, the infrastructure already exists. 
Processes, contrary to threads are expensive to create. You're creating a new city from scratch, that take time and resources.

Thread creation: ~microseconds 
Process creation: ~milliseconds 
That's a 1000x difference in startup cost.



### 3. Communication - Easy vs Structured

```python
# Threads — share variables directly (but dangerously) 
shared_counter = 0 # Both threads see this 

# Processes — must use explicit channels 
# Queue: like a mailbox between countries 
# Pipe: like a telephone line between countries
```


### 4. True Parallelism - No vs Yes

Threads in Python: Thread A runs → passes GIL→ Thread B runs → passes GIL → Thread A runs  Fast alternation. FEELS parallel. IS NOT parallel. 

Processes in Python:
Process A runs on Core 1 
Process B runs on Core 2 ← SAME CLOCK TICK ACTUALLY parallel.



|Aspect|Threads|Processes|
