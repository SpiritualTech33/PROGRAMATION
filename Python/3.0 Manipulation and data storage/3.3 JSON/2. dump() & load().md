dump is the way that we have to write a object of python or a python data structure, to a json data structure.

conceptual example and syntax:
```python
import json

data = {'name': 'yamil', 'age': 25, 'favorite_philosophy': 'oriental'}

with open ("example.json", 'w', encoding='utf-8') as file:
	json.dump(data, file, indent=4)
	

#IMPORTANT!!!!
#The indent=4 parameter is key for human readability
```
What I'm doing here?
I'm taking the data structure called "data" and I'm saving it directly in a json file. 


The step by step process be like:
	Take a python object (Dictionary/List)
	Converts to JSON
	Writes it directly to a file

Think about dump like the way you have to write in JSON files


Now, let's move to load()
load is the way that python gives us to READ JSON files

Conceptual example and syntax:
```python
#Imagine that whe are going to use the file created in past example

import json

with open ("example.json", 'r', encoding='utf-8') as file:
	data = json.load(file)

#Now, we have a variable called data, that contains the data structure created before

print(data['name']) #The result would be 'yamil'
```

What I'm doing here?
simple, I'm taking a JSON file, and converting it into python objects. 

The step by step process would be like:
	Read JSON from file
	Converts to a python object (Dictionary/List)
	Return a python object

Think about load like the way that we have to read a JSON file in python.


The pattern
```python

# WRITE to file 
json.dump(python_object, file_object) 


# READ from file 
python_object = json.load(file_object)
```


|Method|Input|Output|
|---|---|---|
|`dump()`|Python + file|Writes to file|
|`dumps()`|Python|Returns string|
|`load()`|File|Returns Python|
|`loads()`|String|Returns Python|
