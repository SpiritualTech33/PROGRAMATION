These two always work with strings. Not files, strings, that's why they have an S at the end. It's an S from string.



dumps() takes a python data structure and transform it into a JSON string.

# Syntax:

```python
import json

data = {'name': 'yamil', 'age': 25, 'favorite_philosophy': 'oriental'}

json_string = json.dumps(data)
print(json_string)
#OUTPUT: {"name": "Yamil", "age": 25}'
```

What I'm doing here?
Transforming the dictionary data into a single string.

The step by step process would be:
	Takes Python object (dict/list)
	Converts to JSON format
	Returns a STRING** (not a file)





Now loads() is the inverse process. Takes a JSON string and transform it into a python data structure. 

# Syntax:

```python
import json

json_string = '{'name': 'yamil', 'age': 25, 'favorite_philosophy': 'oriental'}'

data = json.loads(json_string)
print(data['name']) #'yami'
print(type(data)) # < class 'dict' >  #Now, the string becomes a dictionary
```

What I'm doing here?
I'm transforming a JSON string into a python object.

the step by step process would be like:
	Takes JSON string
	Converts to Python object
	Returns dict/list**


The pattern:

```python
# Python → JSON string 
json_string = json.dumps(python_object) 

# JSON string → Python 
python_object = json.loads(json_string)
```


|Method|Input|Output|
|---|---|---|
|`dump()`|Python + file|Writes to file|
|`dumps()`|Python|Returns string|
|`load()`|File|Returns Python|
|`loads()`|String|Returns Python|
