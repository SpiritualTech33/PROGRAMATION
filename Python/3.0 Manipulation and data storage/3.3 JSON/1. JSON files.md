The name JSON = JavaScript Object Notation

The structure:
JSON files, are data files structured similar as python dictionaries.
- A text format for storing **structured data**
- Uses **key-value pairs** (like Python dictionaries!)
- Supports **nested structures** (hierarchies, trees)
- **Human-readable AND machine-readable**

### A Simple Example

**Python dictionary:**

```python
persona = {
    'nombre': 'Yamil',
    'edad': 25,
    'ciudad': 'México',
    'habilidades': ['Python', 'Baseball', 'Meditation']
}
```

**Same data in JSON:**

```json
{
    "nombre": "Yamil",
    "edad": 25,
    "ciudad": "México",
    "habilidades": ["Python", "Baseball", "Meditation"]
}
```

Before JSON files, the data exchanging between systems was painful, verbose and really complex. 
### The Philosophy

JSON embodies **simplicity and universality**:

- **Minimal syntax** - only what's essential
- **Native to JavaScript** - every browser understands it
- **Easy to parse** - every language has JSON libraries
- **Self-describing** - the structure IS the schema
