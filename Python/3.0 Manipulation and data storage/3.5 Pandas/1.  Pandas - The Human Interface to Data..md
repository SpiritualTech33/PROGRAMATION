## 1️⃣ Series: The Single Column

A series is a one-dimensional labeled array. You can think of her like a single column in a spreadsheet, a single attribute with multiple observations. 

This is a **Series**:
- **Values**: `[23.5, 25.0, 22.8, 24.1]` (the measurements)
- **Index**: `[Day 1, Day 2, Day 3, Day 4]` (the labels)

The **Series** preserves this relationship between **value and context**.

**EXAMPLE/CREATING A SERIES**
```python
import pandas as pd

# ======================================== 
# CREATING A SERIES 
# ========================================

players_batting_avg = pd.Series(
	data = [0.298, 0.345, 0.411, 0.276],
	index= ['Nelson Cruz', 'Vladimir Guerrero Jr', 'Shohei Ohtani', 'Aaron Judge']
)

print("Series - Player Batting Averages:") 
print(player_batting_averages) 
print("\n")
```
The output will be:

Nelson Cruz    0.298
Vladimir Guerrero Jr    0.345
Shohei Ohtani    0.411
Aaron Judge   0.276

That's a Series, an intimal relation between an index and his value.


## 2️⃣ DataFrame: The Complete Table

Well, we said before the series in a ONE dimensional data structure, and this is a TWO dimensional, but why? because now we have columns with different data types, pretty similar to a SQL table or an excel spreadsheet

**Mental Model:**

```
Imagine a weather station recording multiple measurements:

Date        Temperature    Humidity    Wind_Speed
-------------------------------------------------
2025-01-01     23.5°C        65%         12 km/h
2025-01-02     25.0°C        70%         8 km/h
2025-01-03     22.8°C        68%         15 km/h
2025-01-04     24.1°C        72%         10 km/h
```

Rows (index) --> Each date is singular observation
Columns --> Each measurement type (Temperature, humidity.)
Each column --> It's a series.

Why it exists? I mean, we already have the series, why we need to complicate what is already simple? well, because in the real world, one single thing can have multiple attributes. For example, a customer can have name, city, address, ID, premium validator, etc. 
The **DataFrame** captures this **multidimensional reality** while keeping everything organized and queryable.

**EXAMPLE/CREATING A DATA FRAME**
```python
import pandas as pd

# ======================================== 
# CREATING A DATAFRAME 
# ========================================

data_frame = pd.DataFrame({
	'player_name': ['Stanton', 'Altuve', 'Henderson', 'Turner', 'Ohtani'],
	'position': ['DH', '2B', 'RF/CF', 'SS', 'DH'],
	'AVG': [0.278, 0.311, 0.247, 0.322, 0.298],
	'home_runs': [27, 21, 13, 28, 51],
	'RBI': [68, 78, 56, 87, 124]
})
```


## 🔍 **THE RELATIONSHIP: Series vs DataFrame**

Here's the key insight:

> **A DataFrame is a COLLECTION of Series that share the same index.**

```
DataFrame = Multiple Series aligned by a common index

Each column in a DataFrame → is a Series
Each row in a DataFrame → is also a Series (when you slice it)
```
