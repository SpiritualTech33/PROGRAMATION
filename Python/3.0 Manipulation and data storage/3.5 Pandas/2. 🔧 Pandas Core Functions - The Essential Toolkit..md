Learning this tools is like practicing the basic postures of kung fu before getting inside a combat. This functions belongs to a phase that in data science is called:
	 EDA - Exploratory Data Analysis

Before moving to describe the functions that we have available on pandas to work with our data, let's talk first about the 4 categories of this functions.

1. 🚪 DATA INGESTION - Getting data INTO Pandas
2. 👁️ DATA INSPECTION - Understanding what you have
3. 📊 DATA DESCRIPTION - Statistical summaries
4. 🔍 DATA QUALITY - Finding problems (missing values, types)

With this said, let' move into the functions, by categories.




## 🚪  DATA INGESTION

`pd.read_csv()`
What it does? simple, takes a CSV file and transform it into a DataFrame in pandas. Is the gateway between files and Data Frames. You're not opening a file, you're taking plane text and transforming it into analyzable structure.
```python
import pandas as pd

books_dataframe = pd.read_csv('books_collection.csv', encoding='utf-8')
#Now, the variable called books_dataframe has the data frame extracted from a CSV file.
```

Philosophical Insight:

> `pd.read_csv()` is the **materialization** of data - it takes abstract information stored on disk and **instantiates** it as a concrete Python object you can manipulate.

## 👁️  DATA INSPECTION - Seeing Your Data

These functions answers to the question, what I'm looking for?

`head() / tail()`
What it does? These are methods that returns a subset of rows, it can be the first rows, using head() or the last ones, using tail() from a Data Frame. They exists, because very often you're going to have Data Frames with thousand of rows, and to check the structure yo don't need to see them all, maybe you only need the first's ones or the last's ones. 
```python
import pandas as pd

# Default: shows first 5 rows 
books_dataframe.head() 

# Custom: shows first 10 rows 
books_dataframe.head(10) 

# Shows last 5 rows 
books_dataframe.tail() 

# Shows last 3 rows 
books_dataframe.tail(3)
```
---

`shape`
Shape is an atribute that returns a tuple with 2 values. The total amount of rows, and how many columns each row has.  This is kye, shape is not a function, is an atribute from the Data Frame. Is the path to know exactly the dimensions of our data. 
```python
# Returns: (number_of_rows, number_of_columns) 

books_dataframe.shape 

# Example output: (1000, 5) 
# Means: 1000 books, 5 attributes per book
```
**Philosophical Insight:**
	 `shape` reveals the **dimensionality** of your data universe. It's the answer to "How big is this reality I'm analyzing?"

---

`columns`
As shape, this is not a function, is an atribute. Returns a list with  the names of all the columns of our data base. It's powerful, cuz allow us to understand semantically what each atribute means.
```python

books_dataframe.columns
# Returns: Index(['title', 'author', 'year', 'price'], dtype='object')
```
**Philosophical Insight:**
	 `columns` is the **ontological inventory** - it tells you what aspects of reality you're capturing for each entity.

---

`dtypes`
This returns a Series with the atribute name and his type.

Data type determines what operations are valid:
	- `int64` / `float64` → Can do math (sum, average, multiply)
	- `object` (string) → Can't do math (would cause error)
	- `datetime64` → Can calculate time differences
	- `bool` → Can do logical operations

```python
# ======================================== 
# CHECK DATA TYPES OF EACH COLUMN 
# ======================================== 

# Mental model: Understanding what kind of operations are possible  on each attribute 

print("Data types in our inventory:") 
print(books_inventory_dataframe.dtypes)
```
**Philosophical Insight:**
	 `dtypes` reveals the **epistemological nature** of each variable - it tells you **how** you can know and manipulate each attribute.

---

`info()`
Is a method that gives you a full overview summary of your DataFrame.
	- Number of rows/columns

# ========================================

## 📊  DATA DESCRIPTION - Statistical Summaries

`describe()`
describe is an statistical profile for numeric columns. 
- count --> How many values
- mean --> Average
- std --> Standard derivation
- min --> The minimum value
- 25% --> 1st quartile
- 50% --> Median (Middle value)
- 75% --> 3rd quartile
- max --> The max value

```python
#Example of statistical summary of numeric columns

print("Statistical profile of books prices and publication years")
statistical_summary = books_inventory_dataframe.describe()
print(statistical_summary)

#Now you can extract specific insights
avg_book_price = stattistical_summary.loc['mean', 'price']
print(f"\nAverage book price: ${average_book_price:.2f}")
```
**Philosophical Insight:**
	`describe()` transforms **raw data into knowledge** - it's the statistical compression of a thousand observations into 8 meaningful numbers.

---

`value_counts()`
A function that counts the frequency of unique values in a column. This reveal categorical distribution, to catch data entry errors, and to observe tendencies or patterns.

```python

print("Top 10 authors by book count in inventory:") 
author_frequency = books_inventory_dataframe['author'].value_counts() 
print(author_frequency.head(10)) 

print("\n") 

print("Genre distribution:") 
genre_frequency = books_inventory_dataframe['genre'].value_counts() 
print(genre_frequency)
```

**Philosophical Insight:**
	`value_counts()` reveals the **distribution of discrete reality** - it shows which categories manifest most frequently in your data universe.

## 🔍  DATA QUALITY - Finding Problems

`isnull()` + `sum()`
this combinations allows to count missing values. isnull() creates a boolean matrix and sum() counts the True values. 
```python
# DETECT AND QUANTIFY MISSING DATA

print("Missing Values per Column")
missing_values_summary = books_inventory_dataframe.isnull().sum()
print(missing_values_summary)

# Calculate percentage of missing data 
total_rows = books_inventory_dataframe.shape[0] 
missing_percentage = (missing_values_summary / total_rows) * 100 

print("Percentage of missing data per column:") 
for column_name, percentage in missing_percentage.items(): 
	if percentage > 0: 
		print(f" {column_name}: {percentage:.2f}% missing")
```
**Philosophical Insight:**
 `isnull()` reveals the **incompleteness of observation** - it shows where our knowledge of reality has gaps.

## 🎯 SYNTHESIS: The Workflow

```python
import pandas as pd

# ===== STEP 1: INGEST DATA =====
books_dataframe = pd.read_csv('books_inventory.csv')



# ====== STEP 2: INITIAL INSPECTION ======

#First, look at data.
print(books_dataframe.head())

#Dataset dimensions
print(f"Rows: {books_dataframe.shape[0]}, Columns: {books_dataframe.shape[1]}")

#Available columns
print(books_dataframe.columns.tolist())





# ====== STEP 3: STRUCTURAL ANALYSIS =====

# Data types and missing values
print(books_dataframe.info())







# ====== STEP 4: STATISTICAL SUMMARY ======

# Numerical statistics
print(books_dataframe.describe())







# ===== STEP 5: CATEGORICAL ANALYSIS =====

# Most common authors
print(books_dataframe['author'].value_counts().head(5))







# ===== STEP 6: DATA QUALITY CHECK ======

# Missing data summary
print(books_dataframe.isnull().sum())




```
