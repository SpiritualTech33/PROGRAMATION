
Split-Apply-Combine is a methodology, actually, is more than that, it's a philosophy focused on the data transformation, that follows a simple pattern. It describes how we analyzes groups in 3 phases. 
1. SPLIT   - Divide data into groups based on criteria
2. APPLY   - Perform operation on each group independently  
3. COMBINE - Merge results back into a single structure

It follows a simplicity principle. It says, most complex questions becomes easy when you ask them from a group at a time.

For example, here i have raw data. I have players, teams and home runs per player.
┌──────────┬─────────┬──────────┐
│ Player   │ Team    │ HomeRuns │
├──────────┼─────────┼──────────┤
│ Ohtani   │ Angels  │ 46       │
│ Judge    │ Yankees │ 62       │
│ Trout    │ Angels  │ 40       │
│ Betts    │ Dodgers │ 35       │
│ Guerrero │ Jays    │ 32       │
└──────────┴─────────┴──────────┘

Now, we split by group. The groups are going to be the teams. That would be the step 1: SPLIT

Angels Group
┌──────────┬──────────┐
│ Player   │ HomeRuns │
├──────────┼──────────┤
│ Ohtani   │ 46       │
│ Trout    │ 40       │
└──────────┴──────────┘

Yankees Group
┌──────────┬──────────┐
│ Player   │ HomeRuns │
├──────────┼──────────┤
│ Judge    │ 62       │
└──────────┴──────────┘

Dodgers Group
┌──────────┬──────────┐
│ Player   │ HomeRuns │
├──────────┼──────────┤
│ Betts    │ 35       │
└──────────┴──────────┘

Jays Group
┌──────────┬──────────┐
│ Player   │ HomeRuns │
├──────────┼──────────┤
│ Guerrero │ 32       │
└──────────┴──────────┘


Then, we move into step 2: APPLY

Angels  →  (46 + 40) / 2 = 43.0
Yankees →  62.0
Dodgers →  35.0
Jays    →  32.0

And lastly, COMBINE:
┌─────────┬──────────────┐
│ Team    │ Avg HomeRuns │
├─────────┼──────────────┤
│ Angels  │ 43.0         │
│ Yankees │ 62.0         │
│ Dodgers │ 35.0         │
│ Jays    │ 32.0         │
└─────────┴──────────────┘

The mental process is pretty simple. We split our data frame into groups, then, we execute operations to each groups, and after that we combine the groups, to have a simpler, but more informative data frame. This is a simple example. But imagine the power of this philosophy when you are working with an enterprise that wants to get te average of the total expenses from specific apartments.  Or imagine you're working on Amazon and need to know what was the total of products sold to Europe. 

This method, Split - Apply - Combine reflects the natural way humans solve problems.
	First we divide the problem in smaller pieces or sub-problems
	Then, solve each problem independently
	And finally integrate the solution into a final answer

 This is:

- **Divide and Conquer** (algorithms)
- **Reductionism** (scientific method)
- **Modular thinking** (software architecture)
- **Wu Wei** (Taoist principle: work with natural divisions, not against them)

**You're not just learning Pandas. You're learning a way of thinking.**
