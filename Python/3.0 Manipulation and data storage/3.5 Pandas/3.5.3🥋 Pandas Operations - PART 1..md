# .loc[] & .iloc[]

When we talk about pandas, where talking about data. And, as we know, very often we want to select and filter specific data. Maybe we have a books data frame, and we want to know how many books have been written in Germany. Or we can have a customers data frame, and we want to know how much of them spent more than 100 USD the last month buying in our store. With this on mind, is very clear why pandas give us tools to select, and filter info. There's several methods, and that's exactly what we are going to study now.

First of all, we can select data with the name of his column. And select multiples names in a single query.
```python
df["age"]
#This ones returns a series. Because is a one dimensional data structure

df[["age", "salary"]]
#This ones returns a Data Frame

```
But this is pretty basic, and very often is not gonna be enough.


There's a fundamental question, how do i identify what i want? and for that case, we have two answers.
	`.loc[]` ---> Meaning based (semantic)
	&
	`.iloc[]` ---> Position based (Numerical)

This methods reflects two ways to think.
`.loc[]` --> "I want the info from the column called 'players' "
`.iloc[]` --> I want the player from the second row.
This is the bases, but, let's dive deep into it.

---


## 🔍 `.loc[]` - Label-Based Selection

This is a label - indexer. Allows you to recollect info using index and labels. So you can use column names and rows index to find info.

**SYNTAX**
```python
# General Syntax

dataframe.loc[row_selection, column_selection]

# Both are optional
# If you omit columns, you get all columns.
# If you omit rows you get al the rows of the column you choose.
```

**COMPLETE EXAMPLE**
```python
import pandas as pd
import numpy as np

# ===== Create Samples Data Frame =====

mlb_datadrame = pd.DataFrame[{
	'player_name': ['Judge', 'Ohtani', 'Harper', 'Betts'],
	'team': ['Yankees', 'Dodgers', 'Phillies', 'Dodgers'],
	'AVG': [0.349, 0.298, 0.277, 0.301],
	'home_runs': [56, 51, 27, 29],
	'RBI': [102, 111, 87, 93]
}]

# ORIGINAL DATA FRAME
print(mlb_dataframe)



# ===== SELECTING SINGLE ROW BY INDEX =====

first_player_data = mlb_dataframe.loc[0]
print(fisrt_player_data)

# What happens here: You choose the row with index 0
# But you didn't specify wich column
# So python choose the row 0 and all his columns




# ===== SELECTING SINGLE COLUMN =====

all_players_name = mlb_dataframe.loc[:, 'player_name']
print(third_player_name)

# Here you choose the column player_name
# But you didn't specify the row or rows
# So pythons returns all the rows from that column



# ===== SELECTING SINGLE CELL (row + column) =====

third_player_name = mlb_dataframe.loc[2, 'player_name']
print(third_player_name)

# Here you choose the row with index 2
# And the column of his name
# So python returns the name of the row with index 2


# ===== SELECTING MULTIPLE ROWS =====

selected_players = mlb_dataframe.loc[0, 2, 3]
print(selected_players)

# Here you choose the rows with index 0, 2 and 3
# And you didn't specify the columns
# So python returns all the columns from the selected rows.



# ===== SELECTING MULTIPLE COLUMNS =====

names_and_hr = mlb_dataframe.loc[:, 'player_name', 'home_runs']
print(names_and_hr)

# Here you choose the columns name and home runs
# But ypu didn't specify row
# So python returns all the rows, choosing only the player name and his home runs.




# ===== SELECTING ROWS AND COLUMNS TOGETHER =====

specific_data = mlb_dataframe.loc[
	[0,3],
	['player_name', 'team']
]
print(specific_data)

# Here you choose the rows 0 and 3
# And select the name and the team
# So Python will return the name of the player and the team from the rows 0 and 3.

```

## 📍 `.iloc[]` - Position-Based Selection

This one is a position base indexer. You select the data by index position. The index of a row and from a column.

**Key Insight:**
	`.iloc[]` doesn't care about labels. It only cares about **position**.
	Even if your index is `['a', 'b', 'c']`, `.iloc[0]` still gets the first row.

**SYNTAX**
```python
# General syntax
# Same as .loc[] but using integers

dataframe.iloc[row_position, column_position]

# Remember: Python uses 0-based indexing 
# First row = 0, second row = 1, etc.
```


**COMPLETE EXAMPLE**
```python
import pandas as pd
import numpy as np

# ===== Create Samples Data Frame =====

mlb_datadrame = pd.DataFrame[{
	'player_name': ['Judge', 'Ohtani', 'Harper', 'Betts'],
	'team': ['Yankees', 'Dodgers', 'Phillies', 'Dodgers'],
	'AVG': [0.349, 0.298, 0.277, 0.301],
	'home_runs': [56, 51, 27, 29],
	'RBI': [102, 111, 87, 93]
}]

# WE GONNA USE THE SAME DATA FRAME


# ===== SELECTING FISRT ROW =====
# Mental model: Give me the fisrt row, regardless of its label

first_row = mlb_dataframe.iloc[0]
# Here you choose the first row using his index



# ===== SELECTING FIRST COLUMN =====
# Mental model: Give me the fisrt column, i don't care about the name 

first_column = mlb_dataframe.iloc[:, 0]
# Here you choose the first column, using his index



# ===== SELECTING SEPECIFIC CELL =====

specific_cell = mlb_dataframe.iloc[0, 1]
# Here ypu choose the index from the first row and the index of the second column.



# ===== SELECTING RANGE OF ROWS =====

first_trhee_rows = mlb_dataframe.iloc[0:3]
# Here you choose from the row 0 up to row 2.
# Remember that the value of range doesn't includes in the range.
# So if you put range up to 6, you will recive only till row 5



# ===== SELECTING RANGE OF COLUMNS =====

first_two_columns = mlb_dataframe.iloc[:, 0:2]

# Here you choose from the column with index 0 up to index 1.
# Remember, the 2 is not included in the range.
#And observe how you skip rows with the colon and a comma.\



# ===== SELECTING A SUBSET =====
```

|You know column names | `.loc[]` | Semantic clarity - code is self-documenting | 
| Working with specific indices | `.loc[]` | Your index has meaning (dates, IDs, etc.) | 
| Slicing by position | `.iloc[]` | "First 10 rows", "Last 5 columns" | 
| Numerical computation loops | `.iloc[]` | Iterating with integers is natural | 
|Production ML pipelines | `.iloc[]` | Column positions are stable | 
|Exploratory analysis | `.loc[]` | Easier to read and understand |
