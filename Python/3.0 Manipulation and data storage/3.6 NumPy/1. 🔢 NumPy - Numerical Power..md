What is NumPy?
And how is related to AI, scikit Learn and Tensor Flow

Well, NumPy it's a abbreviation for numerical Python. it's a library written on C, which allows us to make faster operations. For example, in python we have lists, that we iterate over them with a for loop. The problem of this is that is a slow process, and the lists in Python allows combined data types. In the other hand, we have the NumPy heart; The ND array. It's similar to a Python list, but with some particularity's. First the data types in an ND array are homogeneous, so, it can only store one data type per array. But this is not the powerful part, actually, sounds like a limitation. The real power of the ND arrays, is the capability to make operations to the entire array in a single move. What this means? 
For example, in Python, if you want to make operations to a list of numbers, you would do this.

```python
numbers = [1, 5, 7, 9, 5, 34]

# Suppose that you want to multiply every number to 2
# I would do this
numbers_multiplied = []
for number in numbers:
	number_multiplied = number * 2
	numbers_multiplied.append(number_multiplied)
	

# But this is slow and tedious
# With NumPy you can do this

import numpy as np

numpy_array = np.array([1, 4, 7, 9, 4])

fast_operation = numpy_array * 2
# And the whole array make the operation.
# It's faster, compact and powerful.
 
```

This doesn't mean python lists are bad, actually are a great tool because they are flexible. But, when we talk about numerical operations, it's way better to use NumPy Arrays.
NumPy is the bridge between Python (human-friendly) and C (machine-speed).

At its core, NumPy provides:
1. **N-dimensional arrays** (the `ndarray` object)
2. **Vectorized operations** (fast mathematical operations on entire arrays)
3. **Linear algebra tools** (matrix operations, eigenvalues, etc.)
4. **Mathematical functions** (sin, cos, exp, log, etc.)

Okey, this is great to know, but, following my philosophical spirit, let's make the most important question. Why this happens? i mean, why NumPy Arrays are way faster than Python lists?

**Python List:** 
``` python
[1, 2, 3, 4, 5] 
# ↓ 
# Each element is a Python object (heavy, flexible) 
# Stored scattered in memory (cache-inefficient) 
# Operations loop in Python (slow interpreter) 
```

**NumPy Array:** 
``` python
([1, 2, 3, 4, 5]) 
# ↓ 
# Contiguous block of memory (cache-efficient) 
# All elements same type (no overhead) 
# Operations in vectorized C code (blazing fast)
```
