# ⚡ Vectorized Operations

What are vectorized operations? And what are *conditional* vectorized operations?

These are the two questions we need to answer — and once you understand them, you'll never want to write a Python loop over an array again.

---

# The Loop Problem

Imagine you have a million temperatures in Celsius and you need to convert them all to Fahrenheit. The instinct of a beginner is to write a loop:

```python
import numpy as np

celsius = np.array([0, 20, 37, 100])

# The slow way — pure Python loop
fahrenheit = []
for temp in celsius:
    fahrenheit.append(temp * 9/5 + 32)

print(fahrenheit)  # [32.0, 68.0, 98.6, 212.0]
```

This works, but it's slow. Python has to interpret each iteration individually, manage the loop variable, and call the arithmetic operations one at a time. For a million elements, this adds up.

---

# The Vectorized Way

A **vectorized operation** applies a mathematical operation to *every element of an array at once*, using low-level C code under the hood. No Python loop. No overhead. Just raw, compiled speed.

```python
import numpy as np

celsius = np.array([0, 20, 37, 100])

# The fast way — vectorized operation
fahrenheit = celsius * 9/5 + 32

print(fahrenheit)  # [ 32.   68.   98.6 212. ]
```

The result is identical, but NumPy processes the entire array as a single mathematical unit. On a million elements, this can be **100x faster** than the loop version.

The key insight is this: **the loop still happens — it just happens in C, not in Python.** NumPy moves the slow part out of Python's reach.

# All the Standard Operations

Every arithmetic operator you know works element-wise on NumPy arrays.

```python
import numpy as np

a = np.array([10, 20, 30, 40])
b = np.array([2,  4,  5,  8])

# Basic arithmetic — all element-wise
print(a + b)    # [12 24 35 48]
print(a - b)    # [ 8 16 25 32]
print(a * b)    # [ 20  80 150 320]
print(a / b)    # [ 5.  5.  6.  5.]
print(a ** 2)   # [ 100  400  900 1600]
print(a % 3)    # [1 2 0 1]

# Math functions — also vectorized
print(np.sqrt(a))   # [3.162 4.472 5.477 6.324]
print(np.log(a))    # [2.302 2.995 3.401 3.688]
print(np.abs(np.array([-1, -2, 3, -4])))  # [1 2 3 4]
```

---

# Comparing Arrays — Vectorized Booleans

Comparison operators also work element-wise, returning an array of `True`/`False` values.

```python
scores = np.array([72, 85, 91, 60, 78, 95])

# Each comparison produces a boolean array
print(scores > 80)   # [False  True  True False False  True]
print(scores == 60)  # [False False False  True False False]

# You can use this directly as an index (boolean indexing from previous lesson)
passing = scores[scores >= 70]
print(passing)  # [72 85 91 78 95]
```

# Conditional Vectorized Operations — `np.where()`

What if you don't just want to *filter*, but to *transform* values based on a condition? That is what `np.where()` is for.

Think of it as a vectorized ternary expression:

> "For every element: if the condition is true, give me `x`. Otherwise, give me `y`."

```python
import numpy as np

# np.where(condition, value_if_true, value_if_false)

scores = np.array([72, 85, 91, 60, 78, 95])

# Label each score as Pass or Fail
labels = np.where(scores >= 70, 'Pass', 'Fail')
print(labels)  # ['Pass' 'Pass' 'Pass' 'Fail' 'Pass' 'Pass']


# Apply a salary bonus — 10% for top performers, 3% for everyone else
salaries = np.array([50000, 80000, 120000, 45000, 90000])
ratings  = np.array([7,     9,     10,     6,     8    ])

adjusted = np.where(ratings >= 9, salaries * 1.10, salaries * 1.03)
print(adjusted)
# [51500. 88000. 132000. 46350. 92700.]
```

---

# Nested `np.where()` — Multiple Conditions

You can nest `np.where()` calls to handle more than two outcomes, the same way you would nest ternary operators.

```python
# Three-tier classification
temperatures = np.array([-5, 15, 22, 35, 42])

climate = np.where(
    temperatures < 0,   'Freezing',
    np.where(
        temperatures < 25, 'Comfortable',
        'Hot'
    )
)

print(climate)
# ['Freezing' 'Comfortable' 'Comfortable' 'Hot' 'Hot']
```

For more than three tiers, `np.select()` is cleaner — but `np.where()` handles the common two-way split beautifully.

# Performance Comparison

```python
import numpy as np
import time

data = np.random.randint(0, 100, size=1_000_000)

# --- Python loop ---
start = time.time()
result_loop = []
for x in data:
    result_loop.append(x * 2 + 1)
loop_time = time.time() - start

# --- Vectorized ---
start = time.time()
result_vec = data * 2 + 1
vec_time = time.time() - start

print(f"Loop:       {loop_time:.3f}s")
print(f"Vectorized: {vec_time:.4f}s")
print(f"Speedup:    {loop_time / vec_time:.0f}x faster")

# Typical output:
# Loop:       0.412s
# Vectorized: 0.002s
# Speedup:    ~200x faster
```

# 🧩 Quick Reference

```python
# Element-wise arithmetic
arr * 2          # multiply every element
arr + other_arr  # add two arrays element-wise
arr ** 2         # square every element
np.sqrt(arr)     # square root of every element

# Element-wise comparison (returns boolean array)
arr > 5
arr == 0
arr != 100

# Conditional transformation
np.where(condition, value_if_true, value_if_false)

# Multi-condition transformation
np.where(cond1, val1, np.where(cond2, val2, default))
```

Vectorization is not just a performance trick — it is a different way of thinking. Instead of "do this to each element," you learn to say "do this to the whole." That shift in perspective is one of the most valuable things NumPy teaches you.
