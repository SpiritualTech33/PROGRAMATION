# 📡 Broadcasting

Broadcasting is NumPy's way of performing operations between arrays of *different shapes* — without copying data, and without you having to manually resize anything.

At first it can feel like magic. But once you understand the rules, it becomes one of the most natural and powerful tools in your NumPy toolkit.

---

# The Problem It Solves

Suppose you have a matrix of exam scores for 4 students across 3 subjects, and you want to add 5 bonus points to *every single score*. Without broadcasting, you might think you need another array of the same shape filled with 5s.

```python
import numpy as np

scores = np.array([
    [72, 85, 91],
    [60, 78, 95],
    [88, 70, 83],
    [55, 92, 76]
])

# Naively, you might think you'd need this:
bonus = np.full((4, 3), 5)
result = scores + bonus

# But NumPy lets you simply write this:
result = scores + 5   # Broadcasting in action

print(result)
# [[ 77  90  96]
#  [ 65  83 100]
#  [ 93  75  88]
#  [ 60  97  81]]
```

NumPy automatically "stretches" the scalar `5` to match the shape of `scores`. No extra memory is used — it's a conceptual stretch, not a physical copy.

---

# The Broadcasting Rules

NumPy compares the shapes of two arrays dimension by dimension, starting from the *trailing* (rightmost) dimension. Two dimensions are compatible if:

1. They are equal, **or**
2. One of them is `1`

If neither condition is met, NumPy raises an error.

```python
# Shape (4, 3) + Shape (3,)
# Trailing dims: 3 == 3  ✅
# Missing dim on right side treated as 1: (1, 3) → (4, 3)  ✅
# Result shape: (4, 3)

scores = np.array([
    [72, 85, 91],
    [60, 78, 95],
    [88, 70, 83],
    [55, 92, 76]
])  # shape (4, 3)

subject_bonus = np.array([5, 10, 0])  # shape (3,)
# Add different bonuses to each subject column

result = scores + subject_bonus
print(result)
# [[ 77  95  91]
#  [ 65  88  95]
#  [ 93  80  83]
#  [ 60 102  76]]
```

The array `[5, 10, 0]` gets broadcast across all 4 rows. Subject 1 gets +5, subject 2 gets +10, subject 3 gets +0.

# Row vs Column Broadcasting

This is where broadcasting's expressiveness really shines. By reshaping a 1D array into a column vector, you can broadcast across rows instead of columns.

```python
import numpy as np

data = np.array([
    [10, 20, 30],
    [40, 50, 60],
    [70, 80, 90]
])  # shape (3, 3)

row_values = np.array([1, 2, 3])      # shape (3,)  → broadcasts across rows
col_values = np.array([[1], [2], [3]])# shape (3,1) → broadcasts across columns

# Broadcast across rows (each column scaled by the same factor)
print(data * row_values)
# [[ 10  40  90]
#  [ 40 100 180]
#  [ 70 160 270]]

# Broadcast across columns (each row scaled by a different factor)
print(data * col_values)
# [[ 10  20  30]
#  [ 80 100 120]
#  [210 240 270]]
```

The `reshape(-1, 1)` trick is the most common way to turn a 1D array into a column vector for broadcasting:

```python
weights = np.array([0.5, 1.0, 1.5])
col_weights = weights.reshape(-1, 1)  # shape (3, 1)
```

# Outer Operations — Building Combination Tables

Broadcasting lets you compute the combination of every element from two 1D arrays, without nested loops.

```python
import numpy as np

prices = np.array([10, 20, 30])        # shape (3,)
quantities = np.array([1, 2, 5, 10])  # shape (4,)

# Reshape prices into a column vector to broadcast against quantities
total_cost = prices.reshape(-1, 1) * quantities
# prices:    shape (3, 1)
# quantities: shape (4,)
# result:    shape (3, 4)

print(total_cost)
# [[  10   20   50  100]
#  [  20   40  100  200]
#  [  30   60  150  300]]
```

This is a multiplication table — built in one line, with no loops.

# When Broadcasting Fails

```python
import numpy as np

a = np.array([[1, 2, 3],    # shape (2, 3)
              [4, 5, 6]])

b = np.array([10, 20])      # shape (2,)

# Trailing dims: 3 vs 2 → neither equal nor 1 → ERROR
try:
    result = a + b
except ValueError as e:
    print(f"Error: {e}")
    # Error: operands could not be broadcast together with shapes (2,3) (2,)

# Fix: reshape b to a column vector (2, 1)
b_col = b.reshape(-1, 1)    # shape (2, 1)
result = a + b_col           # (2,3) + (2,1) → (2,3)  ✅
print(result)
# [[11 12 13]
#  [24 25 26]]
```

When broadcasting fails, the fix is almost always to reshape one of the arrays so the dimensions align correctly.

# 🧩 Quick Reference

```python
# Scalar broadcast — applies to every element
arr + 5
arr * 2.5

# 1D array broadcast over rows of a 2D array
matrix + np.array([a, b, c])   # shape (3,) broadcasts across rows

# 1D array broadcast over columns (reshape first)
matrix + np.array([a, b, c]).reshape(-1, 1)   # shape (3, 1)

# Outer product (combination table)
col.reshape(-1, 1) * row      # shape (n, m)

# Broadcasting rules check
# Compare shapes right-to-left:
# equal → compatible
# one is 1 → compatible (stretches)
# neither → ValueError
```

Broadcasting is NumPy's answer to a deep question: *how do you apply an operation uniformly across a dataset when different parts have different shapes?* The answer is elegant — stretch the smaller shape to fit, conceptually, and let the math flow through. No copies, no loops, no waste.
