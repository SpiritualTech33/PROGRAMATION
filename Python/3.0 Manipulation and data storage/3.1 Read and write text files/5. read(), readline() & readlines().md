This functions are very descriptive by his own names, but they have clue boundaries that we mus respect.

read() reads the entire file and give it back as a single string. Is great for small or medium files and when  you need al the content at once. You should be careful with large files, because they will consume a lot of memory and there's a lot of probabilities that your program crashes. 

```python
with open("sutra.txt", "r") as f:
    content = f.read()  # Returns entire file as ONE string
```

readline() read just one line at a time, includes the \n at the end to leave spaces and return empty strings when the file ends. Its great for having control with iteration or for large files where you don't need all in memory.

```python
with open("journal.txt", "r") as f:
    first_line = f.readline()   # Returns first line (with \n)
    second_line = f.readline()  # Returns second line
    third_line = f.readline()   # Returns third line
```

and lastly but not least, we have readlines(), that reads all the file, and gives us back a list of strings (each element is a line of the text)

```python
with open("practice_log.txt", "r") as f:
    lines = f.readlines()  # Returns ['line1\n', 'line2\n', 'line3\n']
```


🗝️ Core idea
These methods differ only in **how much of the file you bring into memory at once**.


| Method        | Returns       | Memory | Use Case                  |
| ------------- | ------------- | ------ | ------------------------- |
| `read()`      | One string    | High   | Small files, full content |
| `readline()`  | One line      | Low    | Line-by-line processing   |
| `readlines()` | List of lines | High   | Need all lines as list    |
