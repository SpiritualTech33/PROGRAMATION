

These are different methods that we have to interact with text files. Each of them are different and have his own particularities. Let's dive into it.

('r') Read (Receptive Mode)
	**Ontological nature:** Pure observation without alteration
	What it does? this is reading mode, doesn't make changes to the file.
	- Open a file for reading only
	- File must exist, or [[PYTHON]] raises an error
	- Cannot write or modify
	- Cursor starts at the beginning

	When to use?
	- Reading configuration files
	- Loading training data for ML models
	- Analyzing logs
	- Any time you need information but shouldn't alter the source

```python
file = open("sutra.txt", "r")
```


('w') Write (Creator mode)
	**Ontological nature:** Complete manifestation or annihilation
	*What it does?* Creates a new file to write, if the file already exist, delete ALL the data inside of it
	- Opens file for writing
	- If file exists, **ERASES ALL CONTENT** (tabula rasa)
	- If file doesn't exist, creates it
	- Cursor at the beginning of now-empty file
	  
	- *When to use?*
	 - Creating new log files
	- Generating reports from scratch
	- Saving ML model outputs
	- Any time you want a fresh start

```python
file = open("creation.txt", "w")
```


('a') append (Accumulative mode)
	**Ontological nature:** Growth and continuity
	What it does? Open a file for writing, preserving the content, allowing you to add new info.
	- Opens file for writing
	- Preserves existing content
	- Cursor at the END of file
	- New data added after existing data
	- Creates file if it doesn't exist

	When to use?
	- Daily logs (meditation journal, training diary)
	- Continuous data collection
	- Error logs that should preserve history
	- Any scenario where history matters

```python
file = open("journal.txt", "a")
```


('x') Exclusive creation (protective mode)
	**Ontological nature:** Careful manifestation with safeguards
	What it does? Create a new file for writing and fails if the file already exists.
	- Creates a NEW file for writing
	- **FAILS if file already exists**
	- Prevents accidental overwriting
	- Safety mechanism

	When to use?
	- Creating unique files (reports with timestamps)
	- When overwriting would be catastrophic
	- Protecting existing work
	- Ensuring file uniqueness

```python
file = open("sacred_text.txt", "x")
```


('rb') Read Binary (Raw reality mode)
	**Ontological nature:** Direct engagement with pure data
	What it does? Open files in binary mode, and reads pure data in bytes
	- Opens file in binary mode
	- Reads raw bytes, not text
	- No encoding/decoding
	- Returns bytes objects, not strings

	When to use?
	- Images (PNG, JPEG)
	- Audio/Video files
	- Executables
	- Any non-text data
	- When you need exact byte-for-byte accuracy

```python
file = open("image.png", "rb")
```
