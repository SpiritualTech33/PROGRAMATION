The with statement is the recommended way to close the files, rather than using the manual way of the function close(), but why? 
Well, the human are very propense to errors, it's very common for us to forgot about things. Sometimes we forgot a favor our mom told us to do, sometimes we forget special dates, so, what assure us that we are not going to forgot to close the file if you are a distracted mind like me? exactly, the with statement. And not only help us out if we forgot to close the files. There's some times that we actually use close, but an unexpected error occurs before the close line, and our file is now corrupted.

```python
file = open("data.txt", "w") 
file.write("Important data")
#What if an error occurs here?

result = 10 / 0 # Division by zero - program crashes!
file.close() # This line NEVER executes
```

There's when the with statement comes to the rescue

```python
with open("data.txt", "w") as file:
    file.write("Important data")
    result = 10 / 0  # Even if this crashes...
# File is AUTOMATICALLY closed here, no matter what
```

**Without `with`:**

- You must remember to close
- You must handle all error cases
- Human error is possible
- It's like manual memory management in C

**With `with`:**

- Python remembers for you
- Errors are handled automatically
- Impossible to forget
- It's automatic, elegant, **correct**



From "The  ZEN of Python":

- **"Explicit is better than implicit"** - `with` makes scope explicit
- **"Simple is better than complex"** - `with` is simpler than try/finally
- **"Beautiful is better than ugly"** - `with` is elegant
