Let's think about this 2 functions as the ying and yang. But, let's move into parts.
open() is a function that creates a realm between 2 worlds. The ephemeral world of your RAM memory, and the longe-live world of your hard drive. 

**What happens internally:**
1. Python asks the **Operating System**: "I need access to this file"
2. The OS checks: Does this file exist? Do you have permission?
3. If yes, OS creates a **file descriptor** - think of it as a phone line to that file
4. Python wraps this in a **file object** - your interface to that phone line
5. Now you have a **handle** - a reference that lets you read/write

Then, we have close(), the name is pretty descriptive, this function allow us to close the file. But, why is important to close it? Well, the files consumes memory, and if yo don't close the file, you will not set free that memory, the files run the risk of get corrupted, and releases the lock on the file, so the program can use it. But those are superficial reasons. Think about any natural process. If you open a door, you close it, if you open a book to read, you close it when the session is done, if you turn on your laptop, you turn it off when the work session is donde. Is the natural balance open() is Ying (natural, expansive) and close() is Yang (receptive, contractive)

```python
#Opening a file to write in it
file = open("meditation_log.txt", "w")

# Using the file - writing data
file.write("Day 1: 30 minutes of zazen\n") 
file.write("Focus: breath counting\n") 
file.write("State: calm, centered\n")

#Closing the file (crutial step)
file.close()

print("Meditation log saved sucesfully")
```
