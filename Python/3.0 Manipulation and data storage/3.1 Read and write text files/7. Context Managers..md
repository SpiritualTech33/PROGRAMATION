# Context managers
---
What is a Context Manager? (Simple Version)

A **context manager** is like a responsible friend who:

1. **Sets things up for you** (opens the door)
2. **Lets you do your work** (you walk through)
3. **Cleans up after you NO MATTER WHAT** (closes the door, even if you run away)

### The Magic Word: `with`
When you see `with` in Python, you're using a context manager:
```python
with open("file.txt", "r") as f: 
	content = f.read() # File automatically closes here - guaranteed!
```

Why It Matters

**Without context manager (risky):**

- You might forget to close the file
- If an error happens, file stays open
- Computer runs out of resources
- Data might get lost

**With context manager (safe):**

- Python ALWAYS closes the file
- Even if errors happen
- Even if you forget
- Resources are protected

 The Pattern Everywhere

Context managers appear in:

- **Files** - open/close
- **Databases** - connect/disconnect
- **Networks** - start/stop
- **Your life** - meditation begins/ends, practice starts/finishes
