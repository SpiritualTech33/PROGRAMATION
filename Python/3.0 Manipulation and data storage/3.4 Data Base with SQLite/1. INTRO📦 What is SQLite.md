# Relational Data Base
Before talking about anything else, let's think about what a relational data base is. Think in the human consciousness itself. We don't store all the info in one single chaotic space. Our mind create for us categories, memories, emotions. Create connections between all the data is stored in it. 
When i think in philosophy for example, my mind creates a mental image of the subject philosophy, and from that, start's to make small ramifications between subjects (like epistemology, metaphysics or linguistics), between philosophers (like Kant, Plato, Socrates or Eratosthenes), between eras (before Christ, pre reborn, pre science revolution, post WW2, etc.) Our mind naturally creates the reality using concepts and categories to make differences between one thing and another. 
A database is humanity's attempt to give computers this same organizing intelligence.

## The epistemological shift
Databases emerged from a fundamental question: _"How do we KNOW what data we have, WHERE it is, and HOW it relates to other data?"_

In the 1970s, Edgar Codd (at IBM) had a revelation similar to what Buddha had under the Bodhi tree - he saw that **data has inherent structure**, and if we honor that structure, we can query the universe of information with precision.

This became the **Relational Model** - the Tao of data organization.

A relational data base, is a placer where you store information in tables, called relations, where each table represents a category of existence. Let's use the past example to enlighten this. We can have a table called philosophers, another one called philosophies, and another one called eras. Those are different tables, but have certain relationship between them. This is the main structure, the categories of being. From the tables born the rows and the columns. Pretty similar to [[1. CSV files]]. 

The word "Relational" doesn't jus mean stored related data, it's more profound than that. It comes from relational algebra, a system that Edgar Codd borrowed from set theory and first-order logic.
Codd realized that if we organize data into **relations** (mathematical sets with defined structure), we can **query** that data using **logic** instead of navigating through physical storage.


## Rows - Individual instances of being

Each row in your table is a individual instance of that category. Let's keep using philosopher, just because i really love philosophy. 
Philosophers Table: 

fieldnames: Name, Birth year, Death year, main philosophy
Row 1: Plato, 428 BCE, 348 BCE, Idealism 
Row 2: Kant, 1724 CE, 1804 CE, Transcendental Idealism 
Row 3: Socrates, 470 BCE, 399 BCE, Classical
Row 4: Descartes, 1596 CE, 1650 CE, Methodic doubt

Each row is an individual being, that shares similar atributes with all the individuals in the table.


## Columns

Each column represents a category that all the entities in that particular table shares.

- `name` (what they're called)
- `birth_year` (when they entered existence)
- `death_year` (when they left existence)
- `school_of_thought` (their philosophical essence)

## Relationship

This is the core of the data bases, the capability to relate the info stored in it, just like our brain does. When you thin about plato, for example, you don't think only on him, you can think in his relation with Socrates and Aristoteles. You don't thin in plato and epistemology separately, you know that plato thinks epistemologically. 

In a relational database, we use **keys** to create these connections:

- **Primary Key** = The unique identity of each entity (like a soul's signature)
- **Foreign Key** = A reference to another entity's identity (like karmic bonds between concepts)

## ⚖️ What problem does the relational model solve?

**Before relational databases:**

- Data was duplicated everywhere
- Updates caused inconsistencies
- Truth fractured over time

**The relational model enforces:**

- Single source of truth
- Logical consistency
- Referential integrity

It assumes a worldview:

> Reality should not contradict itself.




# SQLite, What is it?

**SQL = Structured Query Language**
At first glance, SQLite is a structured data base engine, that doesn't need any configuration, installation or connection with outside servers, it lives inside your program, and all the data lives in one single file, is simple, but powerful. 
## **The Direct Answer**

**SQLite is a lightweight, self-contained, serverless relational database engine that stores data in a single file and runs directly inside your program.**

It's the **meditation cushion** of databases - simple, portable, requiring nothing external, yet containing everything needed for the practice.

In simple terms:  
👉 SQLite is a **database that lives inside your application**.


## The historical context

SQLite born in 2000 year, from the mind of **D. Richard Hipp**, he was in charge of a US navy project.
He faced a problem, the need of a database that could run on ships. The main problem is that ships, very often found themselves in deep sea, and it's pretty common to lose signal or connection to regular databases (which needs direct connection to the server.) And from this need born SQLite, a data base that lives inside your program. No need of external connection, everything is inside your program.

### **The Mental Dimension**

SQLite is a **library** that gives your Python program the ability to:

- Create structured tables
- Store data persistently (survives program restarts)
- Query data using SQL
- Maintain relationships between data
- Ensure data integrity

It's like giving your program **photographic memory** with **perfect organization**.



# 🔍 Core Characteristics of SQLite

### 1. 🗂️ File-Based Database

- The entire database is stored in **a single file** (e.g., `data.db`)
- Easy to copy, move, back up, or delete

### 2. 🚫 Serverless

- No database server to install or manage
- No background processes
- Your application reads/writes the database directly

### 3. 📦 Zero Configuration

- No setup, no admin user, no ports
- Just link the SQLite library and start using it

### 4. 🧮 Relational & SQL-Based

- Uses **tables, rows, columns**
- Supports standard SQL:


## The praxis

```python
#First yoy import the library
import sqlite3 

# This creates/opens a database file 
connection = sqlite3.connect('my_database.db') 

# This creates a cursor - your "pen" for writing queries 
cursor = connection.cursor() 

# This executes SQL commands 
cursor.execute("CREATE TABLE philosophers (name TEXT, era TEXT)") 

# This saves changes to disk 
connection.commit() 

# This closes the connection 
connection.close()
```
