### WHERE - The Foundation of Discrimination

This command allows you to make distinctions between one or another individual in your data. It's like giving your program discernment, in where part of your table should be making his changes. 

**SYNTAX**
```sql
SELECT columns FROM table_name WHERE condition;
```

**EXAMPLES**
```sql
--Exact match
SELECT * FROM orders WHERE order_destiny = 'Ontario';

--Numeric Comparassion
SELECT * FROM orders WHERE order_total_price >= 100;

--Not equal
SELECT * FROM orders WHERE order_origin != 'UPS Central Package Managment'

```

This is very powerful, because allows us to create our own conditions to obtain the specific data that we need in certain cases. 


### AND - Conjunction of Conditions

This works pretty similar to Python and operator. What this means is that the 2 (or more conditions) must be True. Is actually pretty simple.

**SYNTAX**
```sql
SELECT FROM table_name
WHERE condition1 AND condition2;
```

**EXAMPLES**
```sql
--Both conditions must match
SELECT * FROM harvard_students
WHERE subject = 'Math' AND grade >= 120;

--Multiple ANDs
SELECT * FROM deep_mind_engineers
WHERE rust_experience = TRUE AND tesis_papers > 5 AND backend_experience >= 3;
```

This is importan, because, meanwhile more ANDs we use, more specific info we will get. 


### OR - Disjunction of Conditions

Same as AND, OR works pretty much the same as python's or operator. It says, if condition one or condition two are true, i give back the data. 

**SYNTAX**
```sql
SELECT columns FROM table_name
WHERE condition1 OR condition2;
```

**EXAMPLES**
```sql
-- Either condition matches 

SELECT * FROM philosophers WHERE birth_year < 0 OR school_of_thought = 'Idealism'; 

-- Ancient philosophers OR idealists (or both) 





-- Multiple ORs 

SELECT * FROM philosophers WHERE name = 'Plato' OR name = 'Kant' OR name = 'Socrates';

-- Specific philosophers
```

OR is expanding, each condition allows to accept more results.


### Combining AND and OR

This enters in the category of complex logic, and it's pretty useful to get very specific data.

**EXAMPLES**
```sql
SELECT * FROM spiritual_tech_developers
WHERE (java_script = TRUE OR rust = TRUE) AND experience_years >= 2;
--This means i want people with 2 or more experience years, it doesn't matter if uses java script or rust

--Without the parenthesis will give a differente result
SELECT*FROM spiritual_tech_developers
WHERE java_script = TRUE OR rust = TRUE AND experience_years >= 2;
-- this means (java script) or (rust AND with 2 or more experience years)

```

**Rule:** AND has higher precedence than OR (like multiplication before addition). **Use parentheses for clarity.**

🧠 Rule:

> **Always use parentheses when mixing AND & OR**


### LIKE - Pattern Matching

LIKE allows you to fin info that follows certain patterns.

|Symbol|Meaning|Example|Matches|
|---|---|---|---|
|`%`|Zero or more characters|`'Pla%'`|`"Plato"`, `"Platon"`|
|`_`|Exactly one character|`'_lato'`|`"Plato"`, `"Xlato"`|

**SYNTAX**
```sql
SELECT columns FROM table WHERE column LIKE pattern;
```

**EXAMPLE**
On Spanish, the termination "logia" means the study of something. Biología, Psicología, Fenomenología, etc.
```sql
SELECT * FROM student_interests
WHERE name LIKE '%logia';
```

**MORE EXAMPLES**
```sql
-- Starts with 'Pla' 
SELECT * FROM philosophers WHERE name LIKE 'Pla%'; 
-- Matches: Plato, Platon 

-- Ends with 'es' 
SELECT * FROM philosophers WHERE name LIKE '%es'; 
-- Matches: Socrates, Descartes 

-- Contains 'art' 
SELECT * FROM philosophers WHERE name LIKE '%art%'; 
-- Matches: Descartes, Aristotle 

-- Exactly 5 characters 
SELECT * FROM philosophers WHERE name LIKE '_____'; 
-- Matches: Plato (5 letters) 

-- Second letter is 'l' 
SELECT * FROM philosophers WHERE name LIKE '_l%'; 
-- Matches: Plato
```


🔤 Case Sensitivity (SQLite Detail)

By default:
- `LIKE` is **case-insensitive** for ASCII
- `'alice'` matches `'Alice'`



## **The Mental Model**

Think of filtering as **consciousness focusing**:

- **WHERE** = "Pay attention to..."
- **AND** = "Must have ALL these qualities" (intersection)
- **OR** = "Must have ANY of these qualities" (union)
- **LIKE** = "Similar to this pattern" (fuzzy match)



| Keyword | Meaning             |
| ------- | ------------------- |
| WHERE   | Filter rows         |
| AND     | All conditions true |
| OR      | Any condition true  |
| LIKE    | Pattern match text  |
