# 🎭 CASE Statements

SQL is not just about retrieving data — sometimes you need to *classify* it on the fly. CASE is SQL's `if/elif/else`. It lets you look at a value and decide what label or result to assign it, right inside your query.

Think of it as a translator: you give it raw data, and it hands back meaning.

```sql
-- Mental model: "Read this salary and tell me the tier it belongs to"
SELECT
    name,
    salary,
    CASE
        WHEN salary >= 90000 THEN 'Senior'
        WHEN salary >= 60000 THEN 'Mid-level'
        WHEN salary >= 35000 THEN 'Junior'
        ELSE 'Intern'
    END AS experience_tier
FROM employees;
```

SQL evaluates each `WHEN` condition from top to bottom and returns the first one that is true. Once a match is found, it stops — just like an `elif` chain in Python.

---

# The Two Forms

## 1. Searched CASE (the most common)

This is what we saw above. You write a full condition after each `WHEN`.

```sql
CASE
    WHEN condition_1 THEN result_1
    WHEN condition_2 THEN result_2
    ELSE default_result
END
```

## 2. Simple CASE (comparing one column to fixed values)

When you're matching a single column against specific values, this is more concise.

```sql
SELECT
    name,
    status_code,
    CASE status_code
        WHEN 1 THEN 'Active'
        WHEN 2 THEN 'Suspended'
        WHEN 3 THEN 'Closed'
        ELSE 'Unknown'
    END AS account_status
FROM accounts;
```

# CASE Inside Aggregations — The Real Power

This is where CASE becomes extraordinary. You can combine it with aggregation functions to build *conditional counts and sums* — all in a single query, without multiple passes over the data.

```sql
-- Mental model: "Count how many employees fall into each performance tier
--               in a single scan of the table"
SELECT
    COUNT(CASE WHEN rating >= 9 THEN 1 END) AS top_performers,
    COUNT(CASE WHEN rating BETWEEN 6 AND 8 THEN 1 END) AS solid_performers,
    COUNT(CASE WHEN rating < 6 THEN 1 END)  AS needs_improvement
FROM employee_reviews;
```

When `CASE` finds no match and there's no `ELSE`, it returns `NULL`. And since `COUNT()` ignores `NULL` values, only the matching rows get counted. This pattern is elegant and extremely efficient.

```sql
-- Conditional SUM: total revenue split by region
SELECT
    SUM(CASE WHEN region = 'North' THEN revenue ELSE 0 END) AS north_revenue,
    SUM(CASE WHEN region = 'South' THEN revenue ELSE 0 END) AS south_revenue,
    SUM(CASE WHEN region = 'East'  THEN revenue ELSE 0 END) AS east_revenue
FROM sales;
```

# CASE in ORDER BY — Dynamic Sorting

You can even sort by the result of a CASE expression.

```sql
-- Sort by priority: urgent orders first, then standard, then low
SELECT order_id, order_type, created_at
FROM orders
ORDER BY
    CASE order_type
        WHEN 'urgent'   THEN 1
        WHEN 'standard' THEN 2
        WHEN 'low'      THEN 3
        ELSE 4
    END;
```

# Full Python Example with SQLite

```python
import sqlite3

with sqlite3.connect('empresa.db') as conn:
    cursor = conn.cursor()

    # -------------------------------------------------------
    # EXAMPLE 1: Classify employees into experience tiers
    # -------------------------------------------------------
    print("=" * 60)
    print("Employee Experience Tiers")
    print("=" * 60)

    cursor.execute("""
        SELECT
            name,
            salary,
            CASE
                WHEN salary >= 90000 THEN 'Senior'
                WHEN salary >= 60000 THEN 'Mid-level'
                WHEN salary >= 35000 THEN 'Junior'
                ELSE 'Intern'
            END AS tier
        FROM employees
        ORDER BY salary DESC
    """)

    for name, salary, tier in cursor.fetchall():
        print(f"  {name:<20} ${salary:>8,.0f}   [{tier}]")


    # -------------------------------------------------------
    # EXAMPLE 2: Count employees per tier — all in one query
    # -------------------------------------------------------
    print("\n" + "=" * 60)
    print("Headcount by Tier")
    print("=" * 60)

    cursor.execute("""
        SELECT
            COUNT(CASE WHEN salary >= 90000 THEN 1 END) AS seniors,
            COUNT(CASE WHEN salary BETWEEN 60000 AND 89999 THEN 1 END) AS mid_levels,
            COUNT(CASE WHEN salary BETWEEN 35000 AND 59999 THEN 1 END) AS juniors,
            COUNT(CASE WHEN salary < 35000 THEN 1 END) AS interns
        FROM employees
    """)

    row = cursor.fetchone()
    print(f"  Seniors:    {row[0]}")
    print(f"  Mid-level:  {row[1]}")
    print(f"  Juniors:    {row[2]}")
    print(f"  Interns:    {row[3]}")
```

# 🧩 Quick Reference

```sql
-- Searched CASE (full conditions)
CASE
    WHEN condition THEN result
    WHEN other_condition THEN other_result
    ELSE default
END

-- Simple CASE (single column comparison)
CASE column
    WHEN value_1 THEN result_1
    WHEN value_2 THEN result_2
    ELSE default
END

-- Conditional COUNT (ELSE omitted → NULL → ignored by COUNT)
COUNT(CASE WHEN condition THEN 1 END)

-- Conditional SUM
SUM(CASE WHEN condition THEN value ELSE 0 END)
```

CASE transforms SQL from a retrieval language into a reasoning language. Once you learn to see it that way, you'll find yourself reaching for it constantly.
