# 🔬 Advanced Filtering

Up to this point you've used `WHERE column = value`. That is the foundation. But SQL gives you a much richer vocabulary for expressing conditions. This lesson is about expanding that vocabulary so you can ask the database precise, nuanced questions.

---

# LIKE — Pattern Matching

`LIKE` searches for a pattern inside a text column. It uses two special wildcard characters:

- `%` — matches *any sequence* of characters (including none)
- `_` — matches *exactly one* character

```sql
-- Everyone whose name starts with "A"
SELECT name FROM customers WHERE name LIKE 'A%';

-- Everyone whose name ends with "son"
SELECT name FROM customers WHERE name LIKE '%son';

-- Everyone whose name contains "aria" anywhere
SELECT name FROM customers WHERE name LIKE '%aria%';

-- Names that are exactly 5 characters long and start with "J"
SELECT name FROM customers WHERE name LIKE 'J____';
```

`LIKE` is case-insensitive in SQLite by default for ASCII characters. Keep that in mind when filtering names or emails.

---

# IN — Matching a List

Instead of writing `WHERE country = 'France' OR country = 'Germany' OR country = 'Spain'`, you can use `IN` for a cleaner statement.

```sql
-- Match any value in the list
SELECT name, country
FROM customers
WHERE country IN ('France', 'Germany', 'Spain');

-- The opposite: everyone NOT in those countries
SELECT name, country
FROM customers
WHERE country NOT IN ('France', 'Germany', 'Spain');
```

`IN` also pairs perfectly with subqueries, as you saw in the previous lesson.

---

# BETWEEN — Ranges

`BETWEEN` is inclusive on both ends — it matches values from the lower bound *through* the upper bound.

```sql
-- Products priced between 10 and 50 (10 and 50 are both included)
SELECT name, price
FROM products
WHERE price BETWEEN 10 AND 50;

-- Orders placed in 2024
SELECT order_id, created_at
FROM orders
WHERE created_at BETWEEN '2024-01-01' AND '2024-12-31';

-- The opposite
SELECT name, price
FROM products
WHERE price NOT BETWEEN 10 AND 50;
```

For dates, always use the `'YYYY-MM-DD'` format in SQLite.

---

# IS NULL / IS NOT NULL — Handling Missing Data

`NULL` in SQL is not a value — it is the *absence* of a value. You cannot compare it with `=` or `!=`. You must use `IS NULL` and `IS NOT NULL`.

```sql
-- Customers who never provided a phone number
SELECT name
FROM customers
WHERE phone IS NULL;

-- Customers who did provide one
SELECT name, phone
FROM customers
WHERE phone IS NOT NULL;
```

This is one of the most common mistakes in SQL: writing `WHERE phone = NULL`. It will silently return zero rows because `NULL = NULL` evaluates to unknown, not `TRUE`.

# Combining Conditions: AND, OR, NOT

You can build complex conditions by chaining filters. The order of evaluation matters — use parentheses to be explicit.

```sql
-- AND: both conditions must be true
SELECT name, salary, department
FROM employees
WHERE salary > 60000 AND department = 'Engineering';

-- OR: at least one condition must be true
SELECT name, salary
FROM employees
WHERE salary < 30000 OR salary > 100000;

-- NOT: invert a condition
SELECT name
FROM employees
WHERE NOT department = 'HR';

-- Combining — ALWAYS use parentheses to make precedence explicit
-- Mental model: "seniors in Engineering OR anyone in the AI team"
SELECT name, salary, department
FROM employees
WHERE (salary >= 90000 AND department = 'Engineering')
   OR department = 'AI Research';
```

Without parentheses, `AND` has higher precedence than `OR`, which leads to subtly wrong results. Parentheses cost nothing and prevent silent bugs.

# HAVING vs WHERE — A Critical Distinction

`WHERE` filters *individual rows* **before** grouping.
`HAVING` filters *groups* **after** aggregation.

This distinction is fundamental. If you need to filter based on an aggregated result (`SUM`, `COUNT`, `AVG`...), you must use `HAVING`.

```sql
-- WRONG: you cannot use an aggregate function in WHERE
SELECT department, AVG(salary) AS avg_salary
FROM employees
WHERE AVG(salary) > 70000   -- This will throw an error
GROUP BY department;

-- CORRECT: filter the groups after aggregation with HAVING
SELECT department, AVG(salary) AS avg_salary
FROM employees
GROUP BY department
HAVING AVG(salary) > 70000;
```

You can use both in the same query — `WHERE` to pre-filter rows, `HAVING` to post-filter groups:

```sql
-- Only active employees, and only departments with more than 5 of them
SELECT department, COUNT(*) AS headcount
FROM employees
WHERE status = 'active'          -- filters rows first
GROUP BY department
HAVING COUNT(*) > 5;             -- then filters the groups
```

# Full Python Example

```python
import sqlite3

with sqlite3.connect('empresa.db') as conn:
    cursor = conn.cursor()

    # -------------------------------------------------------
    # LIKE: find employees whose name contains "an"
    # -------------------------------------------------------
    print("=" * 55)
    print("Names containing 'an'")
    print("=" * 55)
    cursor.execute("""
        SELECT name, department
        FROM employees
        WHERE name LIKE '%an%'
        ORDER BY name
    """)
    for row in cursor.fetchall():
        print(f"  {row[0]:<20} {row[1]}")


    # -------------------------------------------------------
    # BETWEEN + IS NOT NULL: salary range, email required
    # -------------------------------------------------------
    print("\n" + "=" * 55)
    print("Mid-range salaries with confirmed email")
    print("=" * 55)
    cursor.execute("""
        SELECT name, salary, email
        FROM employees
        WHERE salary BETWEEN 50000 AND 80000
          AND email IS NOT NULL
        ORDER BY salary DESC
    """)
    for name, salary, email in cursor.fetchall():
        print(f"  {name:<20} ${salary:>8,.0f}   {email}")


    # -------------------------------------------------------
    # HAVING: departments with more than 3 employees
    # -------------------------------------------------------
    print("\n" + "=" * 55)
    print("Departments with more than 3 employees")
    print("=" * 55)
    cursor.execute("""
        SELECT department, COUNT(*) AS headcount
        FROM employees
        WHERE status = 'active'
        GROUP BY department
        HAVING COUNT(*) > 3
        ORDER BY headcount DESC
    """)
    for dept, count in cursor.fetchall():
        print(f"  {dept:<25} {count} employees")
```

# 🧩 Quick Reference

```sql
-- Pattern matching
WHERE name LIKE 'A%'          -- starts with A
WHERE name LIKE '%son'        -- ends with son
WHERE name LIKE '%aria%'      -- contains aria
WHERE name LIKE 'J___'        -- J + exactly 3 chars

-- List membership
WHERE country IN ('France', 'Spain')
WHERE country NOT IN ('France', 'Spain')

-- Range (inclusive)
WHERE price BETWEEN 10 AND 50
WHERE price NOT BETWEEN 10 AND 50

-- Null checks
WHERE phone IS NULL
WHERE phone IS NOT NULL

-- Logical operators (use parentheses!)
WHERE condition_a AND condition_b
WHERE condition_a OR condition_b
WHERE NOT condition_a
WHERE (a AND b) OR (c AND d)

-- HAVING (post-aggregation filter)
GROUP BY column
HAVING AGGREGATE_FUNCTION(column) > value
```

SQL is a conversation with your data. The richer your filtering vocabulary, the more precise and insightful that conversation becomes.
