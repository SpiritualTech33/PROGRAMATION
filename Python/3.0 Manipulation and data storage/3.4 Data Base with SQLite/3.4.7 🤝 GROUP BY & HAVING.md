# 🧠 Big Picture

|Clause|Purpose|
|---|---|
|`GROUP BY`|Divide rows into groups|
|`HAVING`|Filter those groups|
Think of it like this:

> `WHERE` filters rows  
> `HAVING` filters groups


### GROUP BY
What this does, is collect rows that share the same value in a column. It's the way SQL enables categorical thinking. It allows us to store the data in categories.

```sql
SELECT
	category
	COUNT(*) AS total_products
FROM products
GROUP BY category;
```

### **CRITICAL RULE: The SELECT Constraint**

**In a GROUP BY query, your SELECT can only contain:**

1. Columns you're grouping by
2. Aggregate functions

```sql
-- ❌ WRONG: 
SELECT categoria, nombre, AVG(precio) 
FROM productos 
GROUP BY categoria; -- Error: 'nombre' is not in GROUP BY and not aggregated 

-- ✅ CORRECT: 
SELECT categoria, AVG(precio) 
FROM productos 
GROUP BY categoria;
```


### HAVING

HAVING is  a filter that you use after grouping in categories.  It's a way to make conditions in the categories you just create.

### **The Order of Operations:** 

``` sql
1. FROM -- Get the table 
2. WHERE -- Filter individual rows 
3. GROUP BY -- Create groups 
4. HAVING --Filter groups 
5. SELECT -- Choose columns 
6. ORDER BY --Sort results
```

**EXAMPLES**
```sql
-- Show categories with more than 50 products 
SELECT categoria, 
COUNT(*) as total_productos 
FROM productos 
GROUP BY categoria 
HAVING COUNT(*) > 50;

-- Show categories where average price exceeds $100 
SELECT categoria, AVG(precio) as precio_promedio 
FROM productos 
GROUP BY categoria 
HAVING AVG(precio) > 100;
```



# 🎯 Ultra Simple Summary

| Clause   | Filters What?      |
| -------- | ------------------ |
| WHERE    | Individual rows    |
| GROUP BY | Creates categories |
| HAVING   | Filters categories |
