This are several functions that allow us to make mathematical operations with the data stored in our tables. We can make additions, multiplications or calculate the average. Pretty useful when you're trying o filter data or make operations to the data, or make conditionals of certain result.
## 🧠 Big Picture

Aggregation functions:

> Take **multiple rows** → return **one summarized value**


### 🔢 1️⃣ COUNT()

This has a very simple application, count's rows, and returns a single number. A row is a single line of your table.

**BASIC EXAMPLE**
```sql
SELECT COUNT(*) FROM users;
-- This counts all the rows in the table
```

**VARIANTS**
```sql
COUNT(*)         -- counts all rows
COUNT(column)    -- counts non-NULL values
COUNT(DISTINCT column)

--EXAMPLE
SELECT COUNT(email) FROM users;
--This will give how many emails do i have.
--And ignore rows where email is Null
```


### 💰 2️⃣ SUM()

His name i very auto descriptive. Take the values of a column and add numerical values.

imagine this

| id  | Price |
| --- | ----- |
| 1   | 10    |
| 2   | 20    |
| 3   | 30    |
```sql
SELECT SUM(price) FROM products
--The result will be 60
```

⚠️ Ignores NULL values.


### 📈 3️⃣ AVG()

Returns the average of a column
Using the same table as example
```sql
SELECT AVG(price) FROM products;
--Will return 20
```

⚠️ Also ignores NULL values.


### 🔝 4️⃣ MAX()

Returns the highest value of a column.
Works for:
- Numbers
- Dates
- Text (alphabetical order)

```sql
SELECT MAX(price) FROM products;
-- Returns 30
```



### 🔻 5️⃣ MIN()

Returns the smallest value.
Works for:
- Numbers
- Dates
- Text (alphabetical order)

```sql
SELECT MIN(price) FROM products;
--Returns 10
```


# The Five Sacred Aggregations:
---
```sql
-- COUNT: How many?
SELECT COUNT(*) FROM products;
-- Returns: the total number of rows

SELECT COUNT(price) FROM products;  
-- Returns: number of non-NULL prices

-- SUM: What is the total?
SELECT SUM(price) FROM products;
-- Returns: combined total of all prices

-- AVG: What is the balance point?
SELECT AVG(price) FROM products;
-- Returns: arithmetic mean (sum/count)

-- MAX: What is the upper boundary?
SELECT MAX(price) FROM products;
-- Returns: highest value

-- MIN: What is the lower boundary?
SELECT MIN(price) FROM products;
-- Returns: lowest value
```

**EXAMPLE IN CONTEXT**
```sql
CREATE TABLE (
	id INTEGER PRIMARY KEY,
	product REAL,
	price REAL,
	buying_date TEXT,
)

-- Individual observations (what you've been doing): 
SELECT * FROM ventas;
-- Returns: every single sale

--Agreggate

SELECT
	COUNT(*) AS total_sales       --Total of rows
	SUM(price) AS total_price     --The sum of all prices
	AVG(price) AS average_price   -- The avg of prices
	MAX(price) AS highest_value   --The highest value of prices
	MIN(price) AS lowest_value    --The lowest value of prices
```
