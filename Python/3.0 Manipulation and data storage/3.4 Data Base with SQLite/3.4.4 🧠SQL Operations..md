This are the fundamental verbs of data existence, with them, we can create, manipulate and use our data. Are the paths we have to manipulate data.

### CREATE TABLE - The Act of Creation

Is one of the fundamental commands, and his name is very descriptive. It allow us tu build the table, with the structure that we want, the rows that we want. It has creativity freedom. It's the moment when you stop to be a coder, and become a creator.

**SYNTAX**
```sql

CREATE TABLE table_name (
	column 1, data type, constraints,
	column 2, data type, constraints,
);
```

**EXAMPLE**
```sql

CREATE TABLE favorite_philosophers (
	philosopher_id INTEGER PRIMARY KEY AUTOINCREMENT,
	name TEXT NOT NULL,
	school_of_thought TEXT,
	human_legacy TEXT
	books_written INTEGER
);
```

This is importan, because you're not only storing data, you're defining the nature of his own existence. You're defining the structure of your data. 


### INSERT - Birth of Data

This command allow you to create rows on your table. Now, from you pre defined structure, you're creating individuals.

**SYNTAX**
```sql
INSERT INTO table_name (column1, column2....)
VALUES (value1, value2....)
```

**EXAMPLE**
```sql

INSERT INTO favorite_philosophers (name, school_of_thought, human_legacy, books_written);
VALUES ('Plato', 'Idealism', 'He molded the western thought', 36);

-- Shorthand (if providing ALL columns in order):
INSERT INTO favorite_philosophers VALUES (NULL, 'Kant', 'Trascendental idealism', 'Nuomena and Phenomena', 9);
-- NULL lets AUTOINCREMENT assign the ID
```


### SELECT - The Quest for Truth

SELECT is an inquiry that allows you to make questions about what is stored in your database.

**Basic Syntax**
```sql

SELECT column1, column2
FROM table_name
WHERE condition
```

**EXAMPLES**

Get everything.
```sql
SELECT * FROM favorite_philosophers;
-- * means "all columns"
```

Get specific columns
```sql
SELECT name, human_legacy FROM favorite_philosophers;
```

Filter with WHERE
```sql
SELECT name FROM favorite_philosophers
WHERE books_written >= 10;
-- Shows only the philosophers with more than 10 written books.
```

Order results
```sql
SELECT name, books_written FROM favorite_philosophers
ORDER BY books_written DESC;
-- DESC = descending (highest first) 
-- ASC = ascending (default)
```

Count and aggregate:
```sql
SELECT COUNT(*) FROM favorite_philosophers WHERE books_written > 5;
-- It will count the philosophers with more than 5 books

SELECT AVG(books_written) FROM favorite_philosophers;
-- The average of books written by my favorite philosophers
```

🧠 `SELECT` is the **most important** SQL command.



### UPDATE - Transformation

This command allows you to modify the data without destroying it.

**SYNTAX**
```sql
UPDATE table_name
SET column1= value1, column2= value2
WHERE condition
```

**EXAMPLE**
```sql
--Here i update the info of a single column in a particular row.
UPDATE favorite_philosophers
SET books_written = 15
WHERE name = 'Kant'

--Here i can update the entire column (Afecting all rows)
UPDATE favorite_philosophers
SET school_of_thought = 'Metaphysics'
-- I set the school of thought from all th philosophers to metaphysiscs.

```

⚠️ CRITICAL WARNING:
Always use WHERE unless you intentionally want to update everything.


### DELETE - Destruction

This command allow us to delete, it's pretty intuitive, by the name haha.

**SYNTAX**
```sql
DELETE FROM table_name
WHERE condition;

```

**EXAMPLE**
```sql

DELETE FROM favorite_philosophers WHERE name = 'Wittgenstein';
DELETE FROM favorite_philosophers WHERE books_written < 3;
```

⚠️ CRITICAL WARNING:
Always use WHERE unless you want to delete everything.


### JOIN - Uniting Realities

JOIN allows you to connect tables through relationship.

**SETUP**
```sql
CREATE TABLE monasteries (
    monastery_id INTEGER PRIMARY KEY,
    name TEXT,
    country TEXT
);

CREATE TABLE monks (
    monk_id INTEGER PRIMARY KEY,
    name TEXT,
    monastery_id INTEGER,
    FOREIGN KEY (monastery_id) REFERENCES monasteries(monastery_id)
);
```

Inner JOIN syntax
```sql
SELECT monks.name, monasteries.name, monasteries.country
FROM monks
INNER JOIN monasteries ON monks.monastery_id = monasteries.monastery_id;
```
Show monks with their monastery info - only monks who have monastery.


Left JOIN
```sql
SELECT monks.name, monasteries.name
FROM monks
LEFT JOIN monasteries ON monks.monastery_id = monasteries.monastery_id;
```
Show all monks, even if the don't have a monastery - shows NULL if no monastery related to that monk.



| Operation      | Meaning          |

| `JOIN`         | Combine tables   |
