# CSV
--- 
Well, let's begin with the lesson. CSV, means Comma-Separated-Values. That means that are text files, created with rows and columns, separated by commas.

nombre,edad,ciudad
Yamil,25,México
Carlos,30,España
Rosa,28,Argentina

Is the most basic data structure that we can interact with. It's a table encoded in text, is human and machine readable. 

We have a special library, called csv, that give us tools to work with the files in a easier and cleaner way.

```python
import csv

with open("data.csv", "r") as f:
    reader = csv.reader(f)
    for row in reader:
        # Each row is already a list
        print(row)
        

#Why Use the `csv` Library?
#**The Problem CSV Solves
#What if your data contains commas?
nombre,comentario
Yamil,"Me gusta Python, es genial"

#Manual split(",") would break this:
["Yamil", ""Me gusta Python", " es genial""] # WRONG - 3 parts instead of 2
```

The csv library handles this for you. 
- Respects quoted fields
- Handles different delimiters (`;`, `\t`, etc.)
- Deals with edge cases
- **Professional-grade parsing**

The `csv` module is a **protocol enforcer**.

It:

- reads text line by line
    
- applies a delimiter
    
- returns structured sequences
    

It never assumes:

- data types
    
- intentions
    
- truth
    

It only ensures:

> “The structure you declared is respected.”
